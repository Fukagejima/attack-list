## STEP 1：GitHub の準備

### 1-1. GitHubアカウント作成
1. https://github.com を開く
2. 右上の **Sign up** をクリック
3. メールアドレス・パスワード・ユーザー名を入力して登録
4. メール認証を完了させる

### 1-2. Git（バージョン管理ツール）のインストール
1. https://git-scm.com/download/win を開く
2. **64-bit Git for Windows Setup** をダウンロード
3. インストーラーを実行（すべてデフォルトのまま「Next」でOK）
4. インストール完了後、**スタートメニュー**から「Git Bash」を検索して起動
5. 以下を入力してEnter（名前とメールはGitHubに登録したものを使用）

```bash
git config --global user.name "あなたの名前"
git config --global user.email "あなたのメールアドレス"
```

### 1-3. GitHubにリポジトリを作成
1. https://github.com にログイン
2. 右上の **＋** → **New repository** をクリック
3. 以下を入力：
   - **Repository name**: `attack-list-agent`
   - **Private** を選択（社内ツールなので非公開推奨）
4. **Create repository** をクリック
5. 表示されたURLをメモしておく（例：`https://github.com/yourname/attack-list-agent.git`）

## STEP 3：Supabase でデータベースを作成

### 3-1. Supabaseアカウント作成
1. https://supabase.com を開く
2. **Start your project** → GitHubアカウントでサインイン（STEP 1で作ったもの）

### 3-2. プロジェクト作成
1. **New project** をクリック
2. 以下を入力：
   - **Project name**: `attack-list-db`
   - **Database Password**: 強いパスワードを設定（メモしておく）
   - **Region**: `Northeast Asia (Tokyo)` を選択
3. **Create new project** をクリック（1〜2分かかります）

### 3-3. データベースのテーブルを作成
1. 左メニューの **SQL Editor** をクリック
2. 以下のSQLを貼り付けて **Run** をクリック

```sql
-- ユーザーテーブル
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username VARCHAR(50) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    display_name VARCHAR(100),
    role VARCHAR(20) DEFAULT 'member',
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 分析セッションテーブル
CREATE TABLE analysis_sessions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    company_name VARCHAR(255),
    service_name VARCHAR(255),
    strengths TEXT,
    existing_industry VARCHAR(255),
    target_revenue_scale VARCHAR(100),
    companies_per_industry INTEGER DEFAULT 10,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 業界分析結果テーブル
CREATE TABLE industry_results (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id UUID REFERENCES analysis_sessions(id) ON DELETE CASCADE,
    data JSONB NOT NULL,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- アタックリスト結果テーブル
CREATE TABLE attack_list_results (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id UUID REFERENCES analysis_sessions(id) ON DELETE CASCADE,
    selected_industries JSONB,
    data JSONB NOT NULL,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- RLS（行レベルセキュリティ）を無効化（アプリ側で制御するため）
ALTER TABLE users DISABLE ROW LEVEL SECURITY;
ALTER TABLE analysis_sessions DISABLE ROW LEVEL SECURITY;
ALTER TABLE industry_results DISABLE ROW LEVEL SECURITY;
ALTER TABLE attack_list_results DISABLE ROW LEVEL SECURITY;
```

### 3-4. APIキーを取得
1. 左メニューの **Settings** → **API** をクリック
2. 以下の2つをメモする：
   - **Project URL**（例：`https://abcdefgh.supabase.co`）
   - **anon public**（`eyJhbGciOi...` で始まる長い文字列）

## STEP 4：Streamlit Cloud にデプロイ

### 4-1. Streamlit Cloud アカウント作成
1. https://share.streamlit.io を開く
2. **Continue with GitHub** → GitHubアカウントでログイン

### 4-2. アプリをデプロイ
1. **New app** をクリック
2. 以下を設定：
   - **Repository**: `あなたのユーザー名/attack-list-agent`
   - **Branch**: `main`
   - **Main file path**: `app.py`
3. **Advanced settings** を開いて **Secrets** に以下を貼り付ける：

```toml
ANTHROPIC_API_KEY = "sk-ant-api03-あなたのAPIキー"
SUPABASE_URL      = "https://あなたのプロジェクトID.supabase.co"
SUPABASE_KEY      = "eyJhbGci...あなたのanonキー"
APP_USERNAME      = "admin"
APP_PASSWORD      = "password123"
```

4. **Deploy!** をクリック（2〜3分でデプロイ完了）

### 4-3. URLを確認
デプロイ完了後、`https://あなたのアプリ名.streamlit.app` のようなURLが発行されます。
このURLをチームに共有すればすぐに使えます。

## よくあるトラブル

| 症状 | 原因 | 対処 |
|------|------|------|
| ログインできない | Secretsの設定ミス | Streamlit Cloud → Settings → Secrets を確認 |
| DBエラーが出る | SUPABASE_URLやKEYが間違い | Supabase → Settings → API で再確認 |
| push できない | Git認証エラー | GitHubのPersonal Access Token を使う |
| アプリが遅い | 初回アクセス時のコールドスタート | 30秒ほど待つと起動する |

## セキュリティ注意事項

- `.env` ファイルは **絶対にGitHubにpushしない**（.gitignoreで除外済み）
- Supabase の `service_role` キーは使わない（`anon` キーのみ使用）
- 本番運用前にパスワードを必ず変更する
- Streamlit Cloud の Secrets は暗号化されて保存されます
